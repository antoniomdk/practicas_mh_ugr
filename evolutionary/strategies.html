algorithms.evolutionary.strategies.crossover_and_mutate(population, toolbox, cxpb, mutpb)[fuente]¶

Part of an evolutionary algorithm applying only the variation part (crossover and mutation). The modified individuals have their fitness invalidated. The individuals are cloned so the returned population is independent of the input population.

algorithms.evolutionary.strategies.elitism(population, offspring)[fuente]¶

Change the worst individual of the population by the best individual of the offspring.

algorithms.evolutionary.strategies.memetic_strategy(population, num_selected, prob, sort, *args, **kwargs)[fuente]¶

Generic function for memetics strategies.

Parámetros

  • population – list of individuals to apply local search.

  • num_selected – Number of candidates to apply local search

  • prob – Probability of each candidate to mutate

  • sort – if true, it sorts the candidates by fitness.

  • args – Positional arguments to local_search

  • kargs – Keyword arguments to local_search

algorithms.evolutionary.strategies.run(toolbox, population_size, max_evaluations, cxpb, mupb, generational=True, mem_strategy=None)[fuente]¶

Run an evolutionary algorithm until a maximun number of evaluations.

Parámetros

  • toolbox – A Toolbox that contains the evolution operators.

  • population_size – The number of individials of each generation

  • max_evaluations – Max number of fitness function evaluations.

  • cxpb – Crossover probability

  • mupb – Mutation probability

  • generational – if true, it runs a generational strategy. Else, it runs an stationary strategy

  • mem_strategy – The memetic strategy to use after crossover and mutation.
